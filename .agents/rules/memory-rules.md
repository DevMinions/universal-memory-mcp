
# 记忆管理规则 (Universal Memory MCP)

你已连接 `universal-memory` MCP Server，拥有 18 个记忆管理工具。请遵循以下规则自动管理用户记忆。

## 核心行为

### 对话开始时 — 自动召回
当用户发来第一条消息时，如果消息**不是**简单问候或系统命令，立即调用 `memory_recall` 检索相关记忆：
- 用消息核心意图作为 query
- limit 设为 5
- 将召回的记忆纳入你的回复上下文（不需要明确告知用户）

### 对话过程中 — 自动存储
当你在对话中发现以下任何一种信息时，调用 `memory_store` 保存：
- **用户偏好**：`category: "preference"`（如：喜欢的技术、工作习惯、沟通方式）
- **重要事实**：`category: "fact"`（如：项目架构、技术栈、部署环境）
- **关键决策**：`category: "decision"`（如：技术选型、架构方案、工作流程变更）
- **实体信息**：`category: "entity"`（如：项目名、团队成员、服务地址）

### 对话结束时 — 反思总结
当对话涉及较多技术讨论或重要决策时，调用 `memory_reflect` 总结不变式和衍生知识。

### Scope 使用规则
存储记忆时，根据内容选择合适的 scope：

| Scope | 用途 | 示例 |
|-------|------|------|
| `global` | 通用知识、跨项目技能、通用偏好 | "喜欢用 TypeScript"、"服务器 IP 是 xxx" |
| `project:approval` | 审批办公系统 (jzow-flow) | Vue3 组件、审批流程、MySQL 配置 |
| `project:gamedev` | 游戏开发项目 | Godot 引擎、游戏设计、资源管理 |
| `project:memory-mcp` | Universal Memory MCP 项目 | MCP 架构、LanceDB 配置 |
| `personal` | 个人生活、习惯 | 作息时间、饮食偏好 |
| `learning` | 学习笔记 | 技术学习、读书笔记 |

- **不确定时用 `global`**
- recall 时不传 scope 则搜索全部
- 同一条信息不要存多个 scope

## 判断标准

### 什么时候该存储
- 用户明确表达偏好（"我喜欢...", "我习惯...", "以后都用..."）
- 发现新的项目事实（技术栈、部署环境、数据库配置）
- 做出重要决策（选择方案 A 而非 B）
- 遇到并解决了一个重要 bug（记录原因和解决方式）

### 什么时候不该存储
- 临时性的调试信息
- 用户的一次性指令（"帮我格式化这段代码"）
- 已经在记忆中存在的信息
- 聊天寒暄

## 自我改进
遇到重要的经验教训时，调用 `self_improvement_log` 记录：
- type: "learning" — 最佳实践、纠正的认知
- type: "error" — 工具故障、配置问题

## 禁止事项
- **不要**在每次对话都提醒用户"我已帮你存储了记忆"，静默执行即可
- **不要**存储用户明确标记为临时的信息
- **不要**对简单问候、确认词调用 memory_recall
